# TheSwarm Dashboard — Test Plan (100% coverage target)

> Goal: every user-observable behavior on the dashboard is exercised by an automated test. Every fragment, every form, every SSE channel, every error path. Discovered via code scan in May 2026 — see route audit in `docs/CODEMAPS/web-routes.md` (or regenerate via `/update-codemaps`).

## Coverage strategy

Three orthogonal layers — every feature appears in at least the first two:

| Layer | Tooling | Scope |
|-------|---------|-------|
| **L1 unit/route** | `pytest` + `httpx.AsyncClient` against ASGI app | every route returns expected status/template/JSON shape; happy + sad path; HTMX vs full-page rendering |
| **L2 integration** | `pytest` + real SQLite + stubbed Anthropic/GitHub | command/query handlers persist + emit events; SSE broadcasts; HTMX swaps; webhook signature paths |
| **L3 E2E** | Playwright (`tests/e2e/`) at 4 breakpoints (375/768/1280/1920) | golden user flows, real form submit + redirect, SSE update arrives, dark + light if applicable, no console errors, no 4xx/5xx in network panel |

Coverage target per layer: **L1 ≥ 95% line / 100% route**, **L2 100% command+query handler**, **L3 100% golden flows + 1 sad-path per surface**.

Tracker: `pytest --cov=src/theswarm/presentation --cov=src/theswarm/application --cov-fail-under=95`.

### Agent role surfaces (Phases C–L)

For every fragment in this group, the L1 contract is identical and is generated by a parametrized pytest fixture:
- GET fragment renders for valid project, returns 404 for missing project, returns expected HTMX-swap target id.
- Each form POST validates required fields (422 on missing), persists via service, returns updated fragment HTML.

### REST API
Cover every endpoint in `routes/api.py` with a contract test asserting:
- HTTP code on success + failure
- JSON envelope shape
- Pagination params honored where present
- Auth/CSRF if added

### Error & cross-cutting
| # | Coverage |
|---|----------|
| E1 | Generic 404 → JSON `{detail}` for `/api/*`, HTML page for browser routes |
| E2 | 422 → field-level errors surfaced in form (HTMX swap into `.field-error`) |
| E3 | 502 LLM → toast with retry hint and cause (auth/timeout/parse) |
| E4 | 503 GitHub down → banner on cost estimate + sprint create |
| E5 | SSE disconnect → client auto-reconnect + replay last 100 events |
| E6 | Concurrent project edits → optimistic concurrency or last-write-wins documented |
| E7 | XSS — every user-supplied string escaped (project_id, sprint description, secret key_name, memory entry) |
| E8 | CSRF — state-changing POST requires same-origin or token (currently same-origin only — track in TODO) |

### Performance gates (L3)
- Dashboard initial load < 1.5 s LCP at 1280 viewport
- HTMX swap < 300 ms p95 on local CI fixture
- SSE first event delivered < 500 ms after `CycleStarted` emitted

## CI gates
1. `pytest --cov-fail-under=95` (L1+L2)
2. `pytest tests/e2e -p no:asyncio` (L3) on PR + nightly
3. `npx playwright test --reporter=html` artifact uploaded
4. Lighthouse CI on dashboard + project detail with budget
5. `ruff` + `mypy --strict` on `presentation/`
