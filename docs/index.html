Quickstart

TheSwarm requires Python 3.12+, uv, and API keys for Claude and GitHub.

1. Clone and install

git clone https://github.com/jrechet/theswarm.git
cd theswarm
uv sync --dev

2. Set environment variables

# Required for real mode (agents actually call APIs)
export ANTHROPIC_API_KEY="sk-ant-..."
export GITHUB_TOKEN="ghp_..."

# Target repository (owner/repo format)
export SWARM_GITHUB_REPO="yourorg/yourapp"

# Optional: Mattermost chat integration
export SWARM_PO_MATTERMOST_TOKEN="your-bot-token"
export EXTERNAL_URL="https://your-server.com"

3. Validate configuration

uv run python -m theswarm.presentation.cli.main validate

This checks all env vars before you start. Fix any errors before proceeding.

4. Register a project

# Register your repo
uv run python -m theswarm.presentation.cli.main projects add my-app yourorg/yourapp

# Verify
uv run python -m theswarm.presentation.cli.main projects list

5. Run your first cycle

# Full cycle (PO → TechLead → Dev → QA → Report)
uv run python -m theswarm.presentation.cli.main cycle --project my-app

# Or run in the original mode (requires SWARM_GITHUB_REPO)
uv run python -m theswarm --cycle

6. Open the dashboard

# Web dashboard (browser)
uv run python -m theswarm.presentation.cli.main serve
# Open http://localhost:8091

# Terminal dashboard (TUI)
uv run python -m theswarm.presentation.cli.main dashboard

The Four Agents

Each agent is a LangGraph StateGraph. They share state and run sequentially within a cycle.

PO (Product Owner)

Morning: Reads status:backlog issues from GitHub, selects the most impactful ones, labels them status:ready, writes a daily plan.

Evening: Validates the demo report, writes a daily summary, posts to Mattermost.

Chat: Handles DMs via @swarm-po. Natural language intent parsing for creating stories, running cycles, checking status.

TechLead

Breakdown: Takes status:ready user stories and splits them into dev-sized tasks with role:dev labels.

Review: Reviews all open PRs from the Dev agent. Uses Claude to analyze code quality, correctness, and conventions. Approves or requests changes. Merges approved PRs.

Dev

Implements: Picks a status:ready task, clones the repo, creates a branch, calls Claude API to write code, runs the test suite, opens a PR.

Iterates: Up to 5 iterations per cycle. If tests fail, the Dev agent reads errors and tries again. Budget-capped by token limits.

QA

Tests: Writes Playwright E2E tests for new features. Runs the full test suite (unit + E2E). Runs semgrep for security scanning.

Reports: Generates a demo report with story breakdowns, quality gate results, coverage, and cost tracking.

Issue Label State Machine

Issues move through labels as agents process them:

status:backlog → status:ready → status:in-progress → status:review → merged/closed
                  (PO)          (TechLead)        (Dev)           (TechLead)

Dashboards

Web Dashboard

FastAPI + HTMX + Jinja2. Server-rendered, no JavaScript framework. Real-time updates via SSE (Server-Sent Events).

/ (Dashboard)

Project stats, active cycles, live activity feed with SSE auto-updates, cost tracking.

/projects/

List, create, view, delete projects. Framework detection on add.

/cycles/

Cycle history with phase timeline. Trigger new cycles from the UI.

/reports/

Demo reports with story breakdowns, quality gates, diff highlights, learnings.

/health

Health endpoint with DB check, SSE check, uptime. Used by Docker healthcheck.

/api/*

JSON API: projects, dashboard data, cycle details, SSE event stream at /api/events.

# Start the web dashboard
uv run python -m theswarm.presentation.cli.main serve
# Open http://localhost:8091

Terminal Dashboard (TUI)

Built with Textual. Shows four agent panels, budget bars, activity log, project cards. Keyboard-driven: p for projects, q to quit, r to refresh.

uv run python -m theswarm.presentation.cli.main dashboard

Webhook Triggers

Configure a GitHub webhook pointing to POST /webhooks/github to auto-trigger cycles on:

  • Push to the default branch
  • New issue opened
  • Pull request review requested

Architecture

TheSwarm combines the original LangGraph agent pipeline with a Clean Architecture / DDD layer for multi-project management.

Clean Architecture Layers

Presentation CLI (argparse), Web (FastAPI+HTMX+SSE), TUI (Textual). User-facing interfaces.
Application CQRS commands/queries, EventBus, DTOs. ReportGenerator, ImprovementEngine, StartupValidator.
Infrastructure SQLite repos, LocalArtifactStore, CronScheduler, WebhookHandler, GitHubTicketSource.
Domain Frozen dataclasses, Protocol ports. 8 bounded contexts: Projects, Cycles, Agents, Tickets, Memory, Reporting, Scheduling, Chat.

Original Agent Pipeline

The LangGraph-based agents live in src/theswarm/agents/. They use a shared AgentState TypedDict and are orchestrated by cycle.py. Each agent calls the Claude API via tools/claude.py and interacts with GitHub via tools/github.py.

Stub Mode

When SWARM_GITHUB_REPO is not set, all agents run in stub mode. They log what they would do but make no API calls. This is how the test suite works (859 tests, all passing in stub mode).

Deployment

Docker Compose

# Build and run
docker compose up

# Or use the pre-built image
docker compose pull && docker compose up -d

The container listens on port 8091, runs healthchecks against /health, and mounts the host's .claude config for Claude Code access.

Docker Swarm (Production)

TheSwarm deploys on Docker Swarm with Traefik reverse proxy. CI runs tests on push/PR, then CD builds a container image to GHCR and deploys via docker stack up.

# Deploy to swarm
docker stack deploy -c docker-compose.yml theswarm

Traefik routes /swarm path prefix to the container. Mattermost callbacks hit /swarm/mattermost/callback.

Cron Schedule (Headless)

For running cycles without the web dashboard:

# Register project and set schedule
uv run python -m theswarm.presentation.cli.main projects add api yourorg/api
uv run python -m theswarm.presentation.cli.main schedule set api "0 8 * * 1-5"

# Start the server (scheduler runs in-process)
uv run python -m theswarm.presentation.cli.main serve
