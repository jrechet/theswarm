TheSwarm

Autonomous AI dev team. Plain English in, shipped PRs out.

13 role-specialised agents · 4-level autonomy spectrum · full audit trail

The problem

Most "AI coding assistants" are autocomplete in a tab.

Real product work isn't autocomplete. It is:

  • Framing outcomes (PO)
  • Architecture decisions and paved roads (TechLead, Architect)
  • Implementation with TDD discipline (Dev)
  • End-to-end testing and quality gates (QA)
  • Security, SRE, design, docs, release, analytics — all the roles you actually need

A single agent loop cannot replace a team. So we built a team.

The pitch

Describe a sprint. The swarm plans, builds, reviews, tests, ships, reports.

Roles — portfolio-level

Architect

Paved-road rules, portfolio ADRs, direction briefs. Rules block merges across all projects until satisfied.

Chief of Staff

Routes incoming chat to the right role, enforces budget policies, runs onboarding checklist, archives.

Scout

External intel: CVEs, releases, papers, competitors. Tags + clusters into briefs for the PO.

Three roles operate above any single project. They make portfolio-wide decisions and feed every project-level role.

The daily cycle

MorningPOPicks status:backlog, labels status:ready, daily plan
BreakdownTechLeadSplits user stories into role:dev sub-tasks
Dev loop ×5DevPicks ready task, implements, tests, opens PR
Review loopTechLeadApprove/request-changes, merge approved PRs
QAQAPlaywright e2e, unit, semgrep, demo screenshots, gates
EveningPOValidates demo report, writes daily report

Autonomous mode: --autonomous loops the cycle until backlog is empty or --max-cycles hit. Recovers past transient failures.

Autonomy spectrum

Per-(project, role) gating. Defaults are conservative — opt in deliberately.

LevelBehaviour
manualHuman-initiated only. Agent does not act.
assistedAgent proposes; human confirms every step.
supervisedAgent acts; human reviews before merge.
autonomousAgent acts and ships unless blocked.

Override at /projects/{id}/autonomy. Every action is logged and irreversible — that is why higher autonomy is opt-in.

Portfolio knowledge

Refactor programs

Coordinate refactors across multiple projects ("move every API to v2 auth"). Proposed → active → completed; opt-in per project.

Semantic memory

Opt-in retrieval. Tag + substring search. Portfolio-wide and per-project. Each role decides what to write.

Prompt library

Versioned prompts with full create/update/deprecate/restore audit. Diff before promoting.

Three-layer memory: portfolio × project × role. Persona prompts inject role-scoped memory at every call.

Resilience

A swarm that crashes once a day is not a swarm. Sprint G shipped:

  • Per-phase hard timeouts — a hung agent cannot brick the cycle.
  • One-retry on transient Dev errors + orphan cycle reaper.
  • Circuit-breaker on the GitHub client.
  • Pre-flight /health/ready — checks Claude CLI, GitHub, repo perms before launch.
  • Cancel button on running cycles.
  • Tight CLI timeouts + memory bump (1G → 2G).

Architecture

Clean Architecture / DDD. 8 bounded contexts. Frozen dataclasses. Protocol ports.

src/theswarm/
├── domain/         frozen dataclasses, value objects, Protocol ports
├── application/    CQRS commands/queries, EventBus, services, DTOs
├── infrastructure/ SQLite repos, schedulers, webhook, ticket sources
└── presentation/
    ├── cli/        argparse CLI
    ├── web/        FastAPI + HTMX + Jinja2 + SSE
    │   └── routes/ 33 modules — one per role + portfolio surface
    └── tui/        Textual dashboard

Bounded contexts: Projects · Cycles · Agents · Tickets · Memory · Reporting · Scheduling · Chat.

Get started

git clone https://github.com/jrechet/theswarm.git
cd theswarm
uv sync --dev
uv run python -m theswarm validate
uv run python -m theswarm

Then open the dashboard, add a project, type a sprint into the composer, click Run Cycle.

Stub mode (no env vars set) runs the entire pipeline without API calls — for trying it out locally.
